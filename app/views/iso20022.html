ISO 20022 Ecosystem

As a Full-Stack architecture I am trying to build ISO 20022 Ecosystem. ISO 20022 ecosystem will help user to concentrate more on process other than preparing for complex software around it. I have build few dotnet as well as angular components to derive this ecosystem around ISO 20022.

XSDService
The powerful dotnet library based on .Net 6 which transform complex XSD to minimal JSON structure. This library will help user to deconstruct complex XSD to JSON which further help to understand the structure and able to construct more important things around it.
dotnet | .Net 6.0
ngx-iso-form
This form is used to design Angular Reactive Form using any given XSD. This component has been derived using the power of XSDService. The primary use of this UI library is to design ISO 20022 forms dynamically.
Angular | TypeScript
ngx-xml-message
This form is used to design Angular Readonly Form using any given XML/ISO 20022 messages. The primary use of this UI library is to design ISO 20022 or MX readonly forms dynamically.
Angular | TypeScript
MessageServices
The enterprise level software helpful to download ISO 20022 messages from different sources. This window or console service make ease parallel connections to different source like IBM MQ, Kafka, RabbitMQ and Azure Queue and help to publish or subscribe ISO 20022 messages. It has simple JSON configuration and easy installation steps. User can even configure multiple windows service pointing to different configuration file.
dotnet | .Net 6.0 | IBM MQ | RabbitMQ | Kafka | Azure Queue
